# Bring-Up Status Details

Detailed implementation state for Nokia Lumia 520 / `fame` mainline bring-up.

| Area | Status | Current State | Next Work |
| --- | --- | --- | --- |
| Workspace | Candidate | Top-level submodules and notes are seeded; kernel and U-Boot submodules are active local branches. | Commit once reviewed; update submodule pointers as active branches are pushed. |
| Live BootMgr inventory | Historical | `lp-externals` has safe `NOKV` and `NOKT`; known `TYPE=RM-914`, `CTR=059S083`; live `UEFI` now contains U-Boot, not Nokia BootMgr. | Restore BootMgr/FlashApp only when fresh Lumia inventory or FFU recovery is explicitly needed. |
| FFU corpus | Working | Stock RM-914 / `059S083` FFU, emergency package, partition images, and PLAT ACPI artifacts are inventoried under `notes/`. | Keep extracted blobs under ignored `extracted/`; record only metadata and decoded facts. |
| ACPI/PCFG | Working | FFU PLAT DSDT/SSDT are decompiled; display PCFG is extracted and names the 480x800 Teisko DSI panel. | Decode GPIO/resource flags before enabling touch, SDCC3 card-detect, panel reset, or regulators. |
| Kernel | Candidate | `linux/` carries minimal MSM8227 GSBI5 UART, USB1/ULPI PHY, TLMM, SDCC1/eMMC pinctrl, MSM8930-style RPM/QFPROM/TSENS, TLMM-owned PSHOLD, and PM8038 L3/L4/L5/L11 regulator wiring; `qcom_defconfig` includes configfs ACM for the test gadget; the latest image boots to initrd, full `ttyMSM0`, RPM-regulated USB/eMMC supplies, eMMC, UDC, and CDC-ACM gadget bind. | Debug remaining non-fatal USB regulator device-link/voltage warnings and bring up secondary CPU support separately. |
| Kernel boot harness | Working | `./build-linux-fastboot.sh` builds `Image.gz`, Android boot-image v2 DTB payload, and the mkosi/APK mini-initrd into `out/fame/fame-linux-fastboot.img` for non-flashing `fastboot boot` from persistent U-Boot. Latest rebuilt image is packaged with `panic=5`, DTB at `0x88400000`, and ramdisk at `0x88600000`. | Keep using non-flashing `fastboot boot`; refresh artifact hashes after each rebuild. |
| U-Boot | Working | Persistent raw APPSBL U-Boot in `UEFI` reaches UART, initializes USB fastboot and eMMC, supports `reboot`, `flash`, `fetch`, `oem run`, `oem console`, and attempts autoboot before falling back to fastboot. The current kernel boot path sets `fdt_high=0xffffffff` and `initrd_high=0xffffffff` explicitly over fastboot before `fastboot boot`. | Keep destructive fastboot flash/erase/GPT operations disabled by procedure unless explicitly requested for a specific target; defer baking the handoff env into U-Boot. |
| USB/display debug | Working | UART output and U-Boot ChipIdea/ULPI fastboot work; Linux exposes `ci_hdrc.0` under `/sys/class/udc`, the mini-initrd binds CDC-ACM, and the host sees `usb-Nokia_Lumia_520_CDC_ACM_fame-if00`. | Clean up the remaining USB regulator warnings. |
| Display | Hypothesis | FFU PCFG says Teisko 480x800 24bpp DSI panel; simple-framebuffer facts are recorded, but the raw U-Boot Linux path intentionally leaves simplefb disabled because the framebuffer reservation overlaps the ARM kernel low load window. | After UART/UDC, revisit simpledrm/simplefb from a boot path with a proven live framebuffer and non-overlapping placement. |
| Touch | Hypothesis | Android4Lumia says Synaptics; stock ACPI corroborates I2C address `0x4B` plus GPIO11/GPIO52 shape, but the minimal first-boot DTS does not model touch. | Validate controller identity, IRQ flags, reset behavior, and supplies before enabling. |
| Storage | Working | U-Boot initializes SDCC1 as `mmc 0`, reports 8-bit MMC 4.5 `008G92` at 7.3 GiB, reads LBA0, and lists the current live GPT. Linux also enumerates the internal eMMC through SDCC1 with TLMM pinctrl and exposes user, boot0, boot1, and RPMB block devices. | Compare live GPT details with the FFU stock GPT and fix SDCC3/external-SD modeling separately. |
